# プロジェクト・ヘッダーファイル・ディレクトリ (include/)

このディレクトリは、プロジェクトのヘッダーファイルを配置するための場所です。

## ヘッダーファイルとは？

ヘッダーファイルは、複数のソースファイル間で共有される C言語の宣言やマクロ定義を含むファイルです。src フォルダ内にあるプロジェクトのソースファイル（C、C++など）で、Cプリプロセッサディレクティブ #include を使用してこれらのファイルを読み込みます。

## src/main.c での記述例
```C:main.c
#include "header.h"

int main (void)
{
  // プログラムの処理
  ...
}
```

## ヘッダーファイルを使用するメリット
- ヘッダーファイルをインクルードすることは、その内容を必要とする各ソースファイルに直接コピーするのと同じ結果をもたらします。しかし、手動でコピーを行うのは手間がかかり、ミスも発生しやすくなります。

- 一括管理: 関連する宣言が一箇所に集約されます。

- 変更の容易さ: 内容を変更する必要がある場合、ヘッダーファイル一箇所を修正するだけで済みます。次に再コンパイルした際、そのヘッダーを読み込んでいるすべてのプログラムに自動的に新しいバージョンが反映されます。

- 一貫性の維持: ヘッダーファイルを使用することで、「修正漏れ」によるプログラム内の一貫性の欠如というリスクを排除できます。

## 慣習

C言語では、ヘッダーファイルのファイル名は .h で終わる名前にするのが一般的です。

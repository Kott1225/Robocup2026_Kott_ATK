# プロジェクト・テスト・ディレクトリ (`test/`)

このディレクトリは、**PlatformIO Test Runner** とプロジェクトの**ユニットテスト（単体テスト）**を配置するための場所です。

## ユニットテストとは？
ユニットテストは、ソースコードの個々のユニット（1つ以上のMCUプログラムモジュール、関連する制御データ、使用手順、操作手順のセット）が、意図した通りに動作し、使用に適しているかどうかを検証するソフトウェアテスト手法です。

### メリット
* **早期発見:** 開発サイクルの早い段階でバグや問題を見つけることができます。
* **品質向上:** コードの変更が既存の機能に影響を与えていないか（デグレード）を即座に確認できます。
* **設計の改善:** テストしやすいコードを書くことは、結果としてシンプルでクリーンな設計に繋がります。
